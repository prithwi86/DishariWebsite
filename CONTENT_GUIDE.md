# JSON Content Guide

This document explains every JSON file used to control the Dishari website content. All JSON files are generated by the sync script (`npm run sync`) from Cloudinary data — **do not edit the local files manually** as changes will be overwritten.

To update content, edit the source in **Cloudinary** and run `npm run build:sync`.

---

## "Commenting Out" JSON Fields

JSON does not support comments. To temporarily disable a field without deleting it, **prefix the key name with an underscore** (`_`):

```json
{
  "_external_url": "https://zeffy.com/ticketing/abc123",
  "button_text": "Register"
}
```

The website treats any `_`-prefixed key as commented out:
- The prefixed key (`_external_url`) is removed
- The un-prefixed key (`external_url`) is set to `null`
- Your code sees `external_url: null` and skips it

To re-enable the field, simply remove the leading underscore.

> **Note:** This convention is applied automatically by `stripCommentedFields()` in `src/utils/jsonHelper.js` and works recursively across all nested objects and arrays in every JSON file consumed by the site.

## home-page.json

**Cloudinary public_id**: `home-page.json`
**Used by**: Multiple homepage components

This is the **central configuration hub** for the homepage. It defines section header texts, image folder sources, inline data (testimonials), and references to other JSON files.

### Structure

```json
{
  "body": {
    "upcoming_events": {
      "header_text": "Upcoming Events",
      "img_urls": {
        "folder": "Dishari/Upcoming",
        "include_subfolders": true,
        "tag": "carousel",
        "urls": ["...resolved at sync time..."]
      }
    },
    "moments": {
      "header_text": "Moments from Past Events",
      "img_urls": {
        "folder": "Dishari/Moments",
        "include_subfolders": false,
        "tag": "",
        "urls": ["...resolved at sync time..."]
      }
    },
    "sponsors": {
      "header_text": "Our Sponsors",
      "img_urls": {
        "folder": "Dishari/Sponsors",
        "include_subfolders": false,
        "tag": "",
        "urls": ["...resolved at sync time..."]
      }
    },
    "past_events": {
      "header_text": "Past Events",
      "json_file": "past-events.json"
    },
    "testimonials": [
      {
        "name": "Person Name",
        "text": "Testimonial quote text."
      }
    ],
    "press_releases": {
      "header_text": "Press Room",
      "json_file_public_id": "press_release.json"
    }
  }
}
```

### Section Details

#### `upcoming_events`
- **`header_text`** — Heading displayed above the upcoming events carousel on the homepage
- **`img_urls.folder`** — Cloudinary folder to search for carousel images
- **`img_urls.include_subfolders`** — Whether to search subfolders (e.g., per-event folders)
- **`img_urls.tag`** — Only include images with this Cloudinary tag (e.g., `carousel`)
- **`img_urls.urls`** — *Auto-populated by sync* — resolved image URLs sorted by `order-N` tags

**Component**: `UpcomingEventBanner.jsx` — auto-playing image carousel

#### `moments`
- **`header_text`** — Heading for the "Moments from Past Events" photo carousel
- **`img_urls`** — Same config structure; folder typically `Dishari/Moments`

**Component**: `Carousel.jsx` — scrolling image carousel

#### `sponsors`
- **`header_text`** — Heading for the sponsors section
- **`img_urls`** — Same config structure; images synced at 250px width (for small logos)

**Component**: `Sponsors.jsx` — 3D rotating sponsor carousel

#### `past_events`
- **`header_text`** — Heading for the past events section
- **`json_file`** — Filename of the data file generated by `syncPastEvents` (default: `past-events.json`)

**Component**: `PastEvents.jsx` — reads header from here, fetches event data from the referenced json_file

#### `testimonials`
Inline array of testimonial objects. No separate JSON file needed.

| Field  | Type   | Description              |
|--------|--------|--------------------------|
| `name` | string | Person's name            |
| `text` | string | Quote / testimonial text |

**Component**: `Testimonials.jsx` — auto-scrolling testimonial carousel

#### `press_releases`
- **`header_text`** — Heading for the press room section (e.g., "Press Room")
- **`json_file_public_id`** — Cloudinary public_id of the press release JSON file

**Component**: `PressRelease.jsx` — reads header from here, fetches press release data from the referenced JSON

### Image Ordering

Add `order-N` tags to images in Cloudinary to control display sequence:
- `order-1` appears first, `order-2` second, etc.
- Images without an `order-N` tag appear after all ordered images
- Works for all sections: upcoming_events, moments, sponsors

## upcoming-events.json

**Cloudinary public_id**: `upcoming-events.json`
**Used by**: `FutureEvent.jsx`, `Navbar.jsx`, `Events.jsx`

The master list of upcoming events. Stored in Cloudinary as raw JSON — the sync script fetches it, then enriches each event with banner/gallery images from Cloudinary folders.

### Structure

```json
{
  "events": [
    {
      "id": "Picnic2026",
      "order": 1,
      "title": "Dishari Picnic 2026",
      "banner": "",
      "details": {
        "description": ["Paragraph 1", "Paragraph 2"],
        "date": "06/13/2026",
        "time": "11:00 AM",
        "venue": "Hopkinton State Park",
        "address": "164 Cedar St, Hopkinton, MA 01748",
        "img_urls": [],
        "video_urls": [],
        "registrations": [
          {
            "button_text": "Event Registration",
            "external_url": "https://www.zeffy.com/...",
            "internal_url": "",
            "embeded_form": "<div>...<iframe src='...'></iframe></div>"
          }
        ]
      },
      "sub_events": [
        {
          "id": "DANCE_WITH_KUMAR_SHARMA",
          "order": 1,
          "title": "Dance With Kumar Sharma",
          "banner": "",
          "details": {
            "description": [],
            "date": "",
            "time": "",
            "venue": "",
            "address": "",
            "img_urls": [],
            "video_urls": [],
            "registrations": []
          }
        }
      ]
    }
  ]
}
```

### Event Fields

| Field         | Type     | Description                              | Source        |
|---------------|----------|------------------------------------------|---------------|
| `id`          | string   | Unique event ID, matches Cloudinary folder name | Manual (Cloudinary JSON) |
| `order`       | number   | Display order (ascending)                | Manual |
| `title`       | string   | Event display name                       | Manual |
| `banner`      | string   | Banner image URL                         | *Auto-resolved by sync* |
| `details`     | object   | Event details (see below)                | Manual + sync |
| `sub_events`  | array    | Child events (same structure)            | Manual + sync |

### Details Fields

| Field          | Type     | Description                              |
|----------------|----------|------------------------------------------|
| `description`  | string[] | Array of paragraphs. Use `•` prefix for bullets |
| `date`         | string   | Date in `MM/DD/YYYY` format (links to Google Calendar) |
| `time`         | string   | Event time                               |
| `venue`        | string   | Venue name (links to Google Maps)        |
| `address`      | string   | Street address                           |
| `img_urls`     | string[] | Gallery image URLs (*auto-resolved by sync*) |
| `video_urls`   | string[] | Video URLs (YouTube, Facebook, Instagram) |
| `registrations`| array    | Registration button configs              |

### Registration Object

| Field          | Type   | Description                              |
|----------------|--------|------------------------------------------|
| `button_text`  | string | Button label (e.g., "Event Registration")|
| `external_url` | string | Opens in new tab                         |
| `internal_url` | string | Opens in same tab                        |
| `embeded_form` | string | HTML snippet with iframe, shown inline   |

**Priority**: `embeded_form` > `internal_url` > `external_url`

### Sub-Events

Sub-events have the exact same structure as top-level events. They render as 3D glass cards on the event page, between the main event details and the "Event Attractions" gallery.

**Cloudinary folder**: `Dishari/Upcoming/<event_id>/<sub_event_id>/`

Sub-event banners are **excluded** from the parent event's "Event Attractions" gallery.

### Description Formatting

Each array item renders as a `<p>` tag. For bullet points:
```json
"description": [
  "Package includes:",
  "• Three virtual mentoring sessions",
  "• One in-person mentoring session",
  "• Stage performance"
]
```

For line breaks within a single paragraph, use `\n`:
```json
"description": [
  "Line 1\nLine 2\nLine 3"
]
```

## past-events.json

**Generated by**: `syncPastEvents()` (scans Cloudinary folder structure)
**Used by**: `PastEvents.jsx`, `EventGallery.jsx`

### Structure

```json
{
  "events": [
    {
      "id": "1_Poush_Sankranti_2026",
      "title": "Poush Sankranti 2026",
      "orderNo": 1,
      "banner": "https://res.cloudinary.com/...",
      "images": ["https://res.cloudinary.com/...", "..."]
    }
  ],
  "metadata": {
    "source": "Cloudinary",
    "folder": "Dishari/Past Events",
    "total_events": 3,
    "generated_at": "2026-03-29T..."
  }
}
```

### How It's Generated

1. Scans subfolders of `Dishari/Past Events/`
2. Folder names must match `<OrderNo>_<EventName>` (e.g., `1_Poush_Sankranti_2026`)
3. Top 3 events by order number are synced
4. Banner = image tagged with `banner` in the folder (fallback: first image)
5. All other images become the gallery

### Event Fields

| Field    | Type     | Description                       |
|----------|----------|-----------------------------------|
| `id`     | string   | Folder name with spaces → underscores |
| `title`  | string   | Parsed from folder name (after `_`) |
| `orderNo`| number   | Parsed from folder name (before `_`) |
| `banner` | string   | Banner image URL                  |
| `images` | string[] | Gallery image URLs (excludes banner) |

## press_release.json

**Cloudinary public_id**: Configured in `home-page.json` → `body.press_releases.json_file_public_id`
**Used by**: `PressRelease.jsx`, `PressReleasePage.jsx`

### Structure

```json
{
  "press_releases": [
    {
      "id": "pr_001",
      "title": "Press Release Title",
      "date": "2026-01-15",
      "summary": "Brief summary shown on card...",
      "description": "Full text of the press release...",
      "images": [],
      "links": [
        {
          "label": "Download PDF",
          "tag": "pdf_asset",
          "url": ""
        }
      ]
    }
  ]
}
```

### How It's Synced

1. Sync reads public_id from `home-page.json` → `body.press_releases.json_file_public_id`
2. Fetches the JSON from Cloudinary
3. For each press release, scans `Dishari/Press_Release/<id>/` for images
4. Resolves tagged link assets from `Dishari/Press_Release/<id>/links/`

### Fields

| Field        | Type     | Description                       |
|--------------|----------|-----------------------------------|
| `id`         | string   | Unique ID, matches folder name    |
| `title`      | string   | Display title                     |
| `date`       | string   | Publication date (ISO format)     |
| `summary`    | string   | Card preview text                 |
| `description`| string   | Full article text                 |
| `images`     | string[] | *Auto-resolved by sync*           |
| `links`      | array    | Link assets resolved by tag       |

## contact.json

**Cloudinary public_id**: `contact.json`
**Used by**: `Contact.jsx`, `Footer.jsx`

### Structure

```json
{
  "contact": {
    "email": "support@dishariboston.org",
    "phone": "",
    "social_media": {
      "facebook": "https://facebook.com/...",
      "instagram": "https://instagram.com/...",
      "youtube": "https://youtube.com/..."
    },
    "background_img_tag": ["asset", "contact"],
    "background_img_url": ""
  }
}
```

### How It's Synced

1. Fetches `contact.json` from Cloudinary
2. Resolves `background_img_url` by searching for images in `Dishari/Assets/` matching all tags in `background_img_tag`

## about-us.json

**Cloudinary public_id**: `about-us.json`
**Used by**: `About.jsx`

Contains organization description and committee member details. Member photos are auto-resolved from `Dishari/About_Us/` folder by matching filenames to member names.

## video_urls.json

**Cloudinary public_id**: `video_urls.json`
**Used by**: Merged into event data during sync

### Structure

```json
{
  "video_urls": [
    {
      "file_name": "upcoming-events.json",
      "event_id": "Agomoni2026",
      "urls": [
        "https://www.youtube.com/watch?v=...",
        "https://www.facebook.com/reel/..."
      ]
    }
  ]
}
```

### How It Works

After all other syncs complete, `syncVideoUrls` reads each entry and merges the `urls` array into the matching event's `details.video_urls` in the target file. Supported video platforms:
- YouTube (watch, shorts, youtu.be)
- Facebook (reels, videos) — opens in new tab
- Instagram (reels, posts) — embedded

## Quick Reference: Where to Edit What

| I want to change...                | Edit this in Cloudinary                              |
|------------------------------------|------------------------------------------------------|
| Homepage section heading           | `home-page.json` → `body.<section>.header_text`      |
| Carousel image order               | Add `order-N` tags to images in Cloudinary            |
| Add/remove carousel images         | Tag/untag images with `carousel` in `Dishari/Upcoming` |
| Testimonials                       | `home-page.json` → `body.testimonials[]`              |
| Add an upcoming event              | `upcoming-events.json` → `events[]` + create folder  |
| Add a sub-event                    | `upcoming-events.json` → `events[].sub_events[]` + create sub-folder |
| Press room heading                 | `home-page.json` → `body.press_releases.header_text`  |
| Press release content              | `press_release.json` (public_id in home-page.json)    |
| Past events heading                | `home-page.json` → `body.past_events.header_text`     |
| Add a past event                   | Create folder in `Dishari/Past Events/`               |
| Sponsor logos                      | Upload images to `Dishari/Sponsors/`                  |
| Contact info / social links        | `contact.json`                                        |
| Video URLs for events              | `video_urls.json`                                     |
| About us / team members            | `about-us.json` + member photos in `Dishari/About_Us/`|

After any change, run `npm run build:sync` and deploy the updated `dist/` folder.
